display:block

·요소를 블록 레벨로 화면에 표시
        · 크기를 지정하면 화면에 지정한 크기대로 반영
        · 인라인 요소라도 크기가 인지되는 블록 요소로 화면에 표시

display:inline

    ·요소를 인라인 레벨로 화면에 표시
    ·크기를 지정해도 크기가 인식되지 않는 인라인 요소로 표시
    ·블록 요소라도 크기가 인식되지 않음
    · 블록 요소의 자식 요소 중 인라인 성격이 있는 요소를 가운데 배치하기 위해 text-align:center 추가 /

인라인 요소

    ·a,span,input,label,i,b,em,strong

블록요소

    ·h1,h2,h3,h4,h5(제목들)
    ·p(paragraph,문단),ul,ol,dl(리스트)
    ·li(리스트 항목),table(표)
    ·div, header, section, article, aside, footer,nav

display:inline-block

    ·img 요소는 태생적으로 인라인 블록 요소의 성격이 있다.
    ·요소를 인라인 요소와 블록 요소의 특징을 모두 갖도록 표시
    ·크기를 지정하면 지정한 크기가 반영됨
    ·text-align속성을 이용하여 정렬 가능
display:table 
    ·테이블 태그가 아닌 요소를  태그처럼 화면에 표시
    ·기본적으로 
 태그는 블록 요소로써 크기가 인식됨
    
display:table-cell
    ·지정한 요소를 

    
  
    
 태그와 같이 화면에 표시
    ·table-cell 값으로 지정한 요소는 테이블의 와 같이 가로 배치됨
    
    *ul li 메뉴를 가로배치할 때, ul을 100%로 li 3개를 3등분 해 넓이를 차지하고 싶은데, 단어 사이 공백 때문에 요소가 밑에 줄로 내려오는 경우
    ul에 display:table, li에 inline-block 대신 display: table-cell을 사용하여 해결 할 수 있다. 또한 ul태그는 기본적으로 내부공간인 padding이 있어 
    padding값을 0으로 지정하여 브라우저 넓이를 다 쓰기한다. (display.html예제 참고)
    
    
  
display:none

    
    ·none으로 지정한 요소를 화면에 표시하지 않음
    ·visibility:hidden, opacity:0 과 달리 요소 자체가 화면에 표시되지 않음(공간을 차지않음)
    ·반응형이나 클릭 등 이벤트에 따라 요소를 표시하지 않도록 설정 가능
    ·opacity:0; 의 경우 자리에 마우스 커스를 대면 커서가 세로선(글자를 선택할 수 있는 모양)으로 바뀜.
    해당 부분에 글씨가 있는데 투며ㅛㅇ도만 낮춰서 안보이게 한 것이 기 때문에 h4:hover {opacity:1} 로 보이게 할 수 있음.
    그러나 display:none과, visibility hidden을 한요소는 이미 화면에 아예 없는 요소이기 때문에 hover를 할 수 없다.
    이때는 해당 요소들의 부모를 만들고 부모에 hover가 나타났을 때 display:block, visibility:visible로 스면 된다. (display.html예제 참고)
